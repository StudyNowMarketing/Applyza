1. BreadcrumbList — All Inner Pages
Priority: CRITICAL — adds breadcrumb trail to SERP snippets, increasing CTR by 10–20%
ℹ️
Implementation approach: The SEO component already accepts a jsonLd prop. Pass the BreadcrumbList schema alongside any existing jsonLd as an array. In React, pass an array: jsonLd={[existingSchema, breadcrumbSchema]} — the SEO component handles arrays correctly.
BreadcrumbList JSON-LD — Template
Pages: All inner pages. Adapt itemListElement per page.
CRITICAL

Replace the itemListElement array with the actual breadcrumbs for each page. Every item needs a position, name, and item (URL). The last breadcrumb does NOT need an item URL.

// Example: /services/visa-immigration
const breadcrumbSchema = {
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "Home",
      "item": "https://applyza.com"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "Services",
      "item": "https://applyza.com/services"
    },
    {
      "@type": "ListItem",
      "position": 3,
      "name": "Visa & Immigration Support"
      // No "item" for the current/last page
    }
  ]
};

In React, pass to SEO component:

// In VisaImmigration.tsx
<SEO
  title="Student Visa Support | 99% Success Rate | Free Help | Applyza"
  description="..."
  path="/services/visa-immigration"
  jsonLd={breadcrumbSchema}
/>
Quick fix for ALL pages: The SEO component renders a single <script> tag. To pass multiple schemas, wrap them in an array or use the @graph pattern. The current component handles arrays via JSON.stringify(jsonLd) — so pass an array of schema objects and they'll all render in one script tag.

4. Article / BlogPosting — Blog Posts
Priority: HIGH — Article schema enables author bylines in Google, rich snippets, and Google Discover cards
BlogPosting JSON-LD — Dynamic (based on Supabase post data)
Pages: /blog/:slug — add to BlogPost.tsx
HIGH

The BlogPost.tsx page already fetches post data from Supabase. Use that data to populate the schema dynamically. Add the following jsonLd prop to the SEO component in BlogPost.tsx:

// In BlogPost.tsx — add after post data is loaded
const blogPostSchema = post ? {
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "BlogPosting",
      "headline": post.title,
      "description": post.excerpt || post.content.substring(0, 155),
      "image": post.cover_image_url || "https://applyza.com/og-default.png",
      "datePublished": post.published_at || post.created_at,
      "dateModified": post.updated_at || post.created_at,
      "author": {
        "@type": "Person",
        "name": post.author_name || "Applyza Editorial Team",
        // Add author URL once author profile pages exist
        // "url": `https://applyza.com/team/${post.author_slug}`
      },
      "publisher": {
        "@type": "Organization",
        "name": "Applyza",
        "url": "https://applyza.com",
        "logo": {
          "@type": "ImageObject",
          "url": "https://applyza.com/logo.png"
        }
      },
      "mainEntityOfPage": {
        "@type": "WebPage",
        "@id": `https://applyza.com/blog/${post.slug}`
      },
      "url": `https://applyza.com/blog/${post.slug}`,
      "inLanguage": "en-GB"
    },
    {
      "@type": "BreadcrumbList",
      "itemListElement": [
        { "@type": "ListItem", "position": 1, "name": "Home", "item": "https://applyza.com" },
        { "@type": "ListItem", "position": 2, "name": "Study Abroad Guides", "item": "https://applyza.com/blog" },
        { "@type": "ListItem", "position": 3, "name": post.title }
      ]
    }
  ]
} : null;
Important: The blog_posts Supabase table needs an author_name column (or author ID linking to a team table) for named authorship to work. Currently all posts default to "Applyza Team" — this must change for E-E-A-T compliance. See Section 6 (Person schema) for author page requirements.

5. Event Schema — Events Page
Priority: MEDIUM — Event rich results show date, location, and registration in SERP
Event JSON-LD — Dynamic (from Supabase events table)
Pages: /events — add to Events.tsx
MEDIUM 
// In Events.tsx — generate from the `upcoming` array
const eventsSchema = upcoming.length > 0 ? {
  "@context": "https://schema.org",
  "@graph": upcoming.map(event => ({
    "@type": "Event",
    "name": event.title,
    "description": event.description || "Study abroad event hosted by Applyza",
    "startDate": event.date,
    "endDate": event.end_date || event.date,
    "eventStatus": "https://schema.org/EventScheduled",
    "eventAttendanceMode": event.location?.toLowerCase().includes("virtual") || event.location?.toLowerCase().includes("online")
      ? "https://schema.org/OnlineEventAttendanceMode"
      : "https://schema.org/OfflineEventAttendanceMode",
    "location": event.location?.toLowerCase().includes("virtual")
      ? { "@type": "VirtualLocation", "url": "https://applyza.com/events" }
      : { "@type": "Place", "name": event.location },
    "organizer": {
      "@type": "Organization",
      "name": "Applyza",
      "url": "https://applyza.com"
    },
    "url": "https://applyza.com/events",
    "isAccessibleForFree": true
  }))
} : null;
Note: Pass eventsSchema as the jsonLd prop to the SEO component. If eventsSchema is null (no upcoming events), the SEO component will safely skip rendering the script tag.

10. SEO Component Improvements
Missing meta tags in the current SEO.tsx component

The existing SEO component is missing several standard tags. These can be added globally in one edit to src/components/SEO.tsx:

// Add inside the <Helmet> block in SEO.tsx

// Robots — explicit indexing instruction
<meta name="robots" content="index, follow, max-image-preview:large, max-snippet:-1" />

// Open Graph locale + image dimensions
<meta property="og:locale" content="en_GB" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:image:type" content="image/png" />

// Twitter site handle
<meta name="twitter:site" content="@applyzahq" />
<meta name="twitter:creator" content="@applyzahq" />
