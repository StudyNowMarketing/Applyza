🚨 #1 Critical Risk — Client-Side Rendering (CSR)

The entire site is a React SPA with no server-side rendering (SSR) or static pre-rendering. All page titles, meta descriptions, canonical tags, and JSON-LD schema are injected after JavaScript executes via react-helmet-async. While Googlebot can render JavaScript, it does so in a second wave — meaning delays, inconsistencies, and crawl budget inefficiency. For a new domain competing against IDP, SI-UK, and AHZ, this is a structural disadvantage. Recommendation: implement SSR (Next.js) or a pre-rendering layer (Vite SSR / Prerender.io) as a medium-term priority.

⚠ Top 5 Priorities Before Any Copy Work

1. Fix the WhatsApp placeholder number (447000000000) on the Book Consultation page — this is live and live users are seeing a fake number.
2. Fix the About page milestones — all 4 say "2026", making the company look brand new and contradicting the 10+ years stat.
3. Fix the H1 tags on service pages — none contain the target keyword.
4. Add BreadcrumbList schema — breadcrumb UI is already in place on most pages; the structured data is missing.
5. Add Blog to the header navigation — it's the primary SEO content asset and Google can't find it from the nav.
