🔍 Architecture Observations & SEO Opportunities

✅ What's Working

✓
Clean URL patterns — slugs for courses, destinations, blog posts. Human-readable and SEO-friendly.
✓
Strong service sub-pages — each service has its own URL, enabling individual keyword targeting.
✓
Dedicated B2B section — /for-institutions and /for-partners separate the audience journeys.
✓
Schema JSON-LD on homepage — Organisation + WebSite + SearchAction already implemented.
✓
Legal compliance pages — Anti-slavery, accessibility, cookie policy — signals trustworthiness.
✓
Sitemap edge function — auto-generated XML sitemap via Supabase edge function.

⚠ Gaps & Opportunities

⚠
No blog categories — /blog/category/:slug missing. Kills SEO clustering and pillar page strategy.
⚠
No scholarship detail pages — /scholarships/:slug would allow targeting "scholarship for Nigerian students in UK" etc.
⚠
No event detail pages — /events/:slug missing, events aren't indexable individually.
⚠
Blog not in header nav — a major content asset is buried; doesn't signal content authority.
⚠
No /about/team page — E-E-A-T signal. Faces = trust for YMYL (study abroad = financial + life decisions).
⚠
No country-specific landing pages — e.g. /study-in-uk, /study-in-canada. High-volume, low-competition opportunity.
⚠
No /resources hub — guides, checklists, visa info all siloed. A /resources section would build topical authority.
⚠
Services missing dropdown nav — 4 sub-services aren't visible from the nav. Users and crawlers miss them.
